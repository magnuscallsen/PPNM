# Recursive Adaptive Integration

Run everything with

```bash
make
```

The project is C++ plus gnuplot. Python is used only for the optional Task B comparison with `scipy.integrate.quad`; its numeric output is written to data files and plotted by gnuplot.

## Main files

- `main.cpp` — runs all tasks, prints informative tables, writes data files.
- `integration.hpp`, `integration.cpp` — ordinary adaptive integrator, Clenshaw-Curtis integrator, infinite-limit transformations, error function.
- `Makefile` — compiles, runs, optionally runs SciPy comparison, and calls gnuplot.
- `scipy_compare.py` — optional SciPy comparison, writes gnuplot-friendly data.

## Plots generated by gnuplot

- `taskA_erf_plot.png` — Task A: `erf(z)` from the implemented integral representation compared with `std::erf`.
- `taskA_erf1_convergence.png` — Task A/C: log-log plot of `|erf(1)-exact|` vs `acc`, with `eps=0`; also shows estimated error.
- `taskB_clenshaw_curtis_calls.png` — Task B: ordinary adaptive integrator vs Clenshaw-Curtis, number of function evaluations.
- `taskB_infinite_calls.png` — Task B: infinite-limit integral tests and number of calls.
- `taskB_scipy_comparison.png` — Task B: C++ integrator vs Python/SciPy `quad`, number of calls; plotted by gnuplot from data produced by `scipy_compare.py`.
- `taskC_error_quality.png` — Task C: estimated error vs actual error.

## Notes

If SciPy is not installed, `make` still runs. The SciPy data file is filled with `NaN` values, and the gnuplot script still exists.
